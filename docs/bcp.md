# BCP протокол

Bootloader Control Protocol.

BCP – асинхронный протокол, основанный на HDLC. В протоколе поддерживается две сущности: запрос (request) и ответ (response).

## Общие сведения о протоколе

Обмен данными всегда инициируется клиентом (утилитой) по модели один запрос – один ответ.

Порядок байт – Big-endian.

Все используемые поля целочисленные и неотрицательные, если не указано обратное. Выравнивание не используется.

## Структура пакетов

### Структура запроса

| Смещение | Поле | Размер (байт) | Описание |
| --- | --- | --- | --- |
| 0 | SOH | 1 | Start-Of-Frame, всегда 0xAA |
| 1 | command | 1 | Код команды |
| 2 | length | 1 | Длина (в байтах) поля data |
| 3 | data | length | Данные |
| 3 + length | CRC | 2 | CRC с позиции 1 до 2 + length включительно |

### Структура ответа

| Смещение | Поле | Размер (байт) | Описание |
| --- | --- | --- | --- |
| 0 | SOH | 1 | Start-Of-Frame, всегда 0xAA |
| 1 | command | 1 | Код команды |
| 2 | status | 1 | Код статуса |
| 3 | length | 1 | Длина (в байтах) поля data |
| 4 | data | length | Данные |
| 4 + length | CRC | 2 | CRC с позиции 1 до 3 + length включительно |

## Схема пакетов

### BCP_FLASH

#### Запрос

| Смещение в data | Описание | Возможные значения |
| --- | --- | --- |
| 0 | Номер слота, в который загружается прошивка | 1 - 2 |

#### Ответ

Поле data пустое.

### BCP_VERIFY

#### Запрос

| Смещение в data | Описание | Возможные значения |
| --- | --- | --- |
| 0 | Номер верифицируемого слота | 1 - 2 |

#### Ответ

| Смещение в data | Описание | Размер (байт) | Возможные значения |
| --- | --- | --- | --- |
| 0 | Флаг валидности образа. 1, если валидно, иначе – 0 | 1 | 0-1 |
| 1 | Часть версии образа (major) | 1 | 0-255 |
| 2 | Часть версии образа (minor) | 1 | 0-255 |
| 3 | Часть версии образа (patch) | 1 | 0-255 |
| 4 | CRC образа | 4 | 0 - 2^32 |
| 8 | Размер тела образа | 4 | 1 - 2^32 |

### BCP_RUN

#### Запрос

| Смещение в data | Описание | Возможные значения |
| --- | --- | --- |
| 0 | Номер запускаемого слота | 1 - 2 |

#### Ответ

Поле data пустое.

### BCP_VERSION

#### Запрос

Поле data пустое.

#### Ответ

| Смещение в data | Описание | Возможные значения |
| --- | --- | --- |
| 0 | Часть версии загрузчика (major) | 0-255 |
| 1 | Часть версии загрузчика (minor) | 0-255 |
| 2 | Часть версии загрузчика (patch) | 0-255 |

## Таблица кодов статуса

| Код | Значение | Описание |
| --- | --- | --- |
| OK | 0x00 | Команда обработана без ошибок |
| UNKNOWN_COMMAND | 0x01 | Неизвестная команда |
| INVALID_DATA | 0x02 | Некорректный формат поля данных |
| BAD_CRC | 0x03 | CRC, указанный в пакете, не совпадает с рассчитанным на фактических байтах |
| INVALID_SLOT | 0x04 | Номер слота имеет недопустимое значение |
| INTERNAL_ERROR | 0x05 | Внутренняя ошибка загрузчика |

## Список команд

| Название | Код | Описание |
| --- | --- | --- |
| FLASH | 0x01 | Запись бинарного файла в указанный слот |
| VERIFY | 0x02 | Верификация и получение некоторых метаданных образа в указанном слоте |
| RUN | 0x03 | Запуск образа в указанном слоте |
| VERSION | 0x04 | Получение версии загрузчика |
